The built-in proxy only contacts hosts that appear on your allowlist. Check the list before fetching remote data so you know which sites are approved.

Why the proxy is required

Fetching remote data goes through the Quantickle proxy so the app can block unsafe protocols, hide your local network details, and prevent accidental requests to untrusted hosts. The allowlist keeps that proxy limited to destinations you explicitly approve.

See what’s allowed

  • Open the Block/Allowlists tab in the main toolbar.
  • Scroll to Proxy Allowlist to view the read-only host list loaded from the server.
  • If the field is empty or shows an error, the server could not load config/proxy-allowlist.json.

Update the allowlist

  • Edit config/proxy-allowlist.json on the server or set the PROXY_ALLOWLIST environment variable (comma-separated).
  • Use plain hostnames (for example, api.example.com) or wildcards like *.example.com to cover subdomains.
  • Restart the Quantickle server after saving changes so the new entries load.

When it’s checked

  • All requests sent through /api/proxy are compared against the allowlist before leaving the server.
  • Requests to hosts not on the list are rejected for safety—add the host and restart if you need to permit it.
  • Only http:// and https:// targets are allowed; other protocols are blocked.
