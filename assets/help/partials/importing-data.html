Bring spreadsheets or saved graphs into Quantickle without touching code. The importer understands common node-and-edge layouts and keeps your existing graph safe by asking before it replaces anything.

CSV file import

  • Works with a node table followed by an edges table separated by an empty row.
    Valid fieldname synonums are shown in parentheses. Only ID, type and labels are mandatory.
  • Node fields supported are: node_id, node_type (type), node_label (label), node_color (color), node_x (x), node_y (y).
  • Edge fields supported are: source_id (source,from), target_id (target,to), edge_type (type), edge_label (label).

Edges file import

  • Simple source–target pairs, separated by commas or spaces. Missing nodes are created automatically.

HOW:

  • Choose File → Import Data and pick a CSV or .edges file.
  • If a graph is already open, decide whether to Replace it or Merge the new nodes and edges into it.
  • Imported items appear immediately. Use Layouts to tidy the view or open the Data Table to check rows.

CSV pointers

  • Keep node rows first with clear headings (for example: node_id, label, type).
  • Add a header that includes source and target when you switch to edges; everything below is treated as relationships.
  • Edge-only sheets are fine—missing nodes are filled in so every relationship has endpoints.

After you import

  • Use Arrange or your preferred layout to space things out.
  • Rename the graph title in the toolbar if you want to keep multiple imported versions.
  • Save as .qut to keep the structure, styling, and any new edits.
